1. System Architecture
1.1 Overview
The TalentLens backend is a RESTful NestJS API with four endpoints. It uses Supabase as the persistence layer and Gemini 2.0 Flash as the AI scoring engine. There is no authentication, no background queues, and no WebSockets in the MVP. All operations are synchronous and request-scoped.

3. API Endpoints
3.1 POST /jobs
Creates a new job. Returns a unique shareable link.
Request Body
{ "title": "React Developer for fintech app", "description": "We need a developer to build...", "budget_min": 500, "budget_max": 1500, "required_tier": "mid", "client_name": "Acme Inc"
}
Response 201
{ "id": "uuid", "unique_link": "a3f7b91c", "apply_url": "/apply/a3f7b91c", "dashboard_url": "/dashboard/a3f7b91c"
}
3.2 GET /jobs/:link
Returns job details. Used by the freelancer Apply page to display the role. Returns 404 if the link does not exist.
3.3 POST /jobs/:link/apply
Freelancer submits their profile. Triggers the full AI scoring pipeline synchronously and returns the result immediately.

4. Scoring Engine
4.1 Where It Lives
ScoringService lives in src/module/scoring/services/. It is imported into ApplicationsModule via ScoringModule and called directly from ApplicationsService after a profile is submitted. It has no controller and no public endpoints.
4.2 Gemini Prompt
A single structured prompt sent to Gemini 2.0 Flash with temperature 0.1 to ensure consistent, deterministic JSON output:
const prompt = ` You are an expert technical recruiter. Evaluate this freelancer for the role.
JOB TITLE: ${job.title} JOB DESCRIPTION: ${job.description} BUDGET RANGE: $${job.budget_min} - $${job.budget_max}
FREELANCER PROFILE: Name: ${application.freelancer_name} Skills: ${application.skills.join(', ')} Years of Experience: ${application.years_experience} Bio: ${application.bio} Portfolio URLs: ${application.portfolio_urls?.join(', ')} Client Testimonies: ${application.testimonies} Proposed Rate: $${application.proposed_rate}
Return ONLY a valid JSON object. No markdown. No preamble. {
"inferred_tier": "junior" | "mid" | "senior", "fit_score": 0-100, "testimony_score": 0-100, "ai_reasoning": "2-3 sentence plain English explanation", "tier_reasoning": "one sentence explaining tier assignment" }
Tier rules (use evidence only -- never self-declaration): junior: 0-2 years, limited/generic portfolio, no specialisation mid: 2-5 years, clear portfolio, some specialisation senior: 5+ years, deep specialisation, leadership signals
`;

4.3 Price Score Calculation
Computed in ApplicationsService -- never by the AI -- to ensure full consistency:
function computePriceScore(proposed: number, min: number, max: number): number { if (proposed >= min && proposed <= max) return 100; if (proposed < min) { const diff = (min - proposed) / min; return Math.max(0, Math.round(100 - diff * 30)); } const overage = (proposed - max) / max; return Math.max(0, Math.round(100 - overage * 100));
}

4.5 Tier Gate Logic
function checkTierAccess(inferred: string, required: string): boolean { if (required === 'any') return true; if (required === 'junior') return inferred === 'junior'; if (required === 'mid') return ['mid', 'senior'].includes(inferred); if (required === 'senior') return inferred === 'senior'; return false;
}

4.6 Gemini Error Handling
If Gemini fails or returns unparseable JSON, the application is saved with a pending state rather than failing the entire request:
try { const raw = await this.gemini.complete(prompt); const clean = raw.replace(/```json|```/g, '').trim(); return JSON.parse(clean);
} catch (err) { this.logger.error('Gemini scoring failed', err); return { inferred_tier: 'unknown', fit_score: 0, testimony_score: 0, ai_reasoning: 'Scoring pending -- please check back shortly.', tier_reasoning: '' };
}

6. Unique Link Generation
Job links are generated using nanoid -- short, URL-safe, and collision-resistant. UUIDs are intentionally avoided for public-facing links as they are too long to share comfortably.
import { nanoid } from 'nanoid'; const unique_link = nanoid(8); // e.g. 'a3f7b91c'
s Install nanoid v3 (CommonJS compatible): npm install nanoid@3. Version 4+ is ESM-only and will break NestJS without extra config.
